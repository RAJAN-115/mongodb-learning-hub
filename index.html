What is MongoDB?

Open-source, document-oriented NoSQL database management system designed for flexibility, scalability, and performance in handling unstructured or semi-structured data.

Think of it like: MongoDB is like a digital filing cabinet with flexible folders - you can put different types of documents in each folder without worrying about a fixed structure.

üìù CRUD Operations

Create (Insert)

db.users.insertOne({ name: "Alice", age: 30, email: "alice@example.com" });

Inserts a single document into the collection

db.users.insertMany([ { name: "Bob", age: 25 }, { name: "Charlie", age: 35 } ], { ordered: false });

Inserts multiple documents. Use {ordered: false} for unordered inserts.

Read (Find)

db.users.find({ age: { $gt: 25 } }); // Find all users older than 25
db.users.findOne({ name: "Alice" }); // Find first user named Alice

Update

db.users.updateOne( { name: "Alice" }, { $set: { age: 31 } } );

Updates a single document. Use updateMany() for multiple documents.

Delete

db.users.deleteOne({ name: "Alice" }); // Deletes first matching document

Deletes a single document. Use deleteMany() for multiple documents.

üéØ Cursors

Think of it like: Cursors are like reading a book page by page instead of loading the entire book into memory at once.
count() Returns number of documents
db.users.find().count()
limit(n) Limits documents returned
db.users.find().limit(5)
skip(n) Skips specified documents
db.users.find().skip(10)
sort() Sorts results (1: asc, -1: desc)
db.users.find().sort({age: -1})

üóÇÔ∏è Indexes

Memory Hook: INDEX = Instant Navigation for Database EXcellence
Think of it like: Indexes are like the index in a book - they help you quickly find what you're looking for without reading every page.

Benefits of Indexes:

Faster Querying
Efficient Sorting
Improved Aggregation
Multi-field Support

Index Management:

Create Index
db.collection.createIndex({ field: 1 });
List Indexes
db.collection.getIndexes();
Unique Index
db.collection.createIndex({ email: 1 }, { unique: true });
Drop Index
db.collection.dropIndex({ field: 1 });
When NOT to use indexes:
  • On rarely used fields (unnecessary resource consumption)
  • When queries return >60% of documents
  • For very small collections
  • Over-indexing can strain resources

‚ö° Aggregation Framework

Memory Hook: MAGIC Pipeline = Match, Aggregate, Group, Include, Count
Think of it like: Aggregation is like a factory pipeline - data goes through different stages and gets transformed at each step.

Aggregation Pipeline Stages:

$match Filter documents
‚Üí
$group Group by field
‚Üí
$sort Sort results
‚Üí
$project Shape output

Common Pipeline Stages:

$match

Filters documents (like find())

$group

Groups documents by field

$sort

Sorts documents in pipeline

$project

Reshapes documents

$unwind

Deconstructs array fields

$limit

Limits number of documents

Real-world Example:

db.sales.aggregate([ { $match: { date: { $gte: "2024-01-01" } } }, { $group: { _id: "$product", totalSales: { $sum: "$amount" }, count: { $sum: 1 } }}, { $sort: { totalSales: -1 } }, { $limit: 10 } ]);

This pipeline finds sales from 2024, groups by product, calculates totals, sorts by sales amount, and returns top 10 products.

MongoDB Atlas

Fully managed cloud database service for easy deployment, management, and scaling.

Cloud Hosting Auto Scaling Global Clusters Backup & Restore

Setup Process:

  1. Create organization & project
  2. Deploy cluster
  3. Configure database user
  4. Set network access (IP whitelist)
  5. Connect via connection string

MongoDB Compass

Graphical User Interface (GUI) for visual database interaction and management.

Visual Interface Query Builder Schema Analysis Index Management

Key Features:

  • Visual document editing
  • Aggregation pipeline builder
  • Query performance insights
  • Real-time server statistics

MongoDB Drivers

Software libraries that enable applications to connect and interact with MongoDB databases.

Node.js Python Java PHP

Node.js Example:

const { MongoClient } = require('mongodb'); const client = new MongoClient(uri); await client.connect(); const db = client.db('myDatabase'); const collection = db.collection('users');

Mongoose ODM

Object Data Modeling library for Node.js providing schema-based solution to model application data.

Schema Validation Middleware Relationships Type Casting

Mongoose vs Official Driver:

Mongoose: Structured schemas, validation, relationships, middleware
Official Driver: Direct database access, more flexible, less overhead

Mongoose Schema Example:

const userSchema = new mongoose.Schema({ name: { type: String, required: true }, email: { type: String, unique: true }, age: { type: Number, min: 0 } }); const User = mongoose.model('User', userSchema);
